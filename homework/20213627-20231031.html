1.新增概念以及用到的语句

1.关于var和let
    在ES6（ECMAScript 2015）之前，JavaScript中定义变量的关键字只有一个：var。
    var定义的变量是函数作用域（function scope），也就是说，在一个函数内部定义的变量只能在该函数内部被访问。
    如果在函数内部没有使用var关键字或者let、const等新的关键字，则定义的变量默认为全局变量，可以被整个页面的代码所访问。
    而在ES6中引入了两个新的关键字：let和const。let和const定义的变量是块级作用域（block scope），也就是说，在一个代码块内部定义的变量只能在该代码块内部被访问。
    let和const相比，最大的区别在于：let定义的变量可以修改，而const定义的变量是常量，不可修改。
    下面是一些let和var的区别：
        1.let定义的变量是块级作用域，而var定义的变量是函数作用域。
        if (true) {
        var x = 1; // 函数作用域
        let y = 2; // 块级作用域
        }
        console.log(x); // 输出 1
        console.log(y); // 报错：y未定义
        2.let定义的变量不能在同一个作用域内重复定义，而var可以。
        var x = 1;
        var x = 2; // 不会报错
        let y = 1;
        let y = 2; // 报错：Identifier 'y' has already been declared
        3.let定义的变量只能在声明之后使用，而var可以在声明之前使用（但此时其值为undefined）。
        console.log(x); // 输出 undefined
        var x = 1;
        console.log(y); // 报错：y未定义
        let y = 2;
2.关于document.createElement方法
    document.createElement 是 JavaScript DOM API 提供的一个方法，用于在内存中创建一个新的 HTML 元素节点。通过该方法创建的元素节点可以随后被添加到文档中的其他元素内或者成为文档树的一部分。
        语法：
        document.createElement(tagName)
        参数 tagName 是要创建的元素的标签名，比如 div, p, img 等。
        创建元素之后，可以使用其他 DOM 方法来修改元素的属性、样式和内容，然后将其插入到文档中。
        document.createElement 方法非常有用，特别是在动态创建页面元素、处理表单、实现交互功能等方面。可以通过结合其他 DOM 操作方法和事件处理程序来进一步操作这些创建的元素。
        需要注意的是，document.createElement 只是在内存中创建了一个元素节点，并没有将其渲染到页面上。要将创建的元素显示在页面上，需要使用其他方法将其插入到文档树中，
        比如使用 appendChild 方法将新创建的元素添加到已存在的元素内部。
3.关于appendChild()是 DOMAPI 中的一个方法，用于将一个元素节点添加为当前元素的最后一个子节点。
    语法：
    parentElement.appendChild(newChild)
    参数 parentElement 是要添加子节点的父元素节点，而 newChild 是要添加的新子节点。注意，newChild 必须是一个有效的元素节点。
    appendChild() 方法会按照以下步骤执行：
    如果 newChild 已经存在于文档中，那么在调用 appendChild() 方法之后，它会被移除并重新插入到新的位置。
    这意味着，如果你想将一个已存在的元素节点移动到另一个位置，可以使用 appendChild() 方法。
    如果 newChild 是一个文档片段（DocumentFragment），那么将会插入该文档片段的所有子节点，而不是插入文档片段本身。
    如果 newChild 已经是当前元素的子节点，那么 appendChild() 操作将不会产生任何效果。同一个节点不能同时是两个地方的子节点。
    将 newChild 添加为 parentElement 的最后一个子节点。
